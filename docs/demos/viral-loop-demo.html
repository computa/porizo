The fix: web stays a great listen, app owns the rest

You can’t make someone download just to re-play a link. So the download is motivated by things the web can’t give:

A · REPLY

"Make one back for Maya — free." Reciprocity at peak emotion. Web can’t create — only the app can. Two-sided: Maya gets one free too (~$0.07 render cost).

B · KEEPSAKES (app-only)

Web = listen. App = HD download + reveal video + lyric card + library. The recipient who wants to truly keep it must cross over — without ever blocking the listen.

C · CLAIM-TO-KEEP URGENCY

"Link live 7 days — claim to keep forever." Friendly urgency converts the "I’ll just bookmark it" crowd.

+ plumbing

Deferred deep link (AppsFlyer OneLink — already have it) so installers land pre-filled; instant play, no PIN; social proof. Target: recipient→register 10–15% → K ≈ 0.1–0.15.
