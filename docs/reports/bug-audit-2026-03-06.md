## CRITICAL FINDINGS (+10 each)

### Auth & Security

| ID | Location | Description |
|----|----------|-------------|
| AUTH-01 | `routes/auth.js:850` | **Missing `await` on logout token revocation.** `revokeAllRefreshTokensForUser` is fire-and-forget. Stolen refresh tokens remain usable after logout. |
| AUTH-02 | `routes/auth.js:940-941` | **Missing `await` on password reset token revocation.** Both `revokeAllRefreshTokensForUser` and `compromiseAllTokenFamiliesForUser` are fire-and-forget. Old sessions survive password change. |

### Billing & Subscriptions

| ID | Location | Description |
|----|----------|-------------|
| BILL-01 | `subscription-manager.js:1099-1120` | **Google sync never calls `updateEntitlements` for existing subscriptions.** Google renewals and plan changes never refill credits. |
| BILL-02 | `subscription-manager.js:675-753` | **`spendSongInTransaction` has no advisory lock.** Concurrent render requests can double-spend — user with 1 song can render 2. |
| BILL-03 | `subscription-manager.js:765-808` | **`spendPoemInTransaction` has same race condition.** No row-level lock on concurrent poem generation. |
| BILL-04 | `subscription-manager.js:626-639` | **`handleRevocation` sets `credits_balance = newBalance` (song count) instead of 0.** Revoked users retain non-zero credits. |
| BILL-05 | `apple-webhook-handler.js:225-231` | **Missing `appleValidator` silently accepts unsigned webhooks.** `basicDecodeJWS` does zero signature verification. Attacker can forge subscription events. |
| BILL-06 | `apple-webhook-handler.js:756-778` | **`buildValidationFromTxInfo` hardcodes `isRevoked: false`.** Masks revocation state on renewal webhooks. |

### Routes & API

| ID | Location | Description |
|----|----------|-------------|
| API-01 | `routes/story.js:49-57` | **Session hijacking via unclaimed story claim.** Any user can permanently take ownership of a `user_id IS NULL` story session. |
| API-02 | `routes/story.js:843-853` | **Admin impersonation via `x-user-id` header.** `debug_user_id` propagates through `app.inject()` into auth middleware. |
| API-03 | `routes/enrollment.js:130-181` | **Missing authentication on `PUT /storage/upload`.** No verification that the uploader owns the session in the key path. |
| API-04 | `routes/admin.js:1009-1113` | **Path traversal in blend analysis.** `userId`/`trackId` from DB used in `path.join()` without containment check (unlike the `/paths` sibling endpoint). |
| API-05 | `routes/sharing.js:688-697` | **PIN timing oracle in share claim.** Plain `!==` string comparison instead of `crypto.timingSafeEqual` (the web-verify endpoint correctly uses it). |

### Database & Migrations

| ID | Location | Description |
|----|----------|-------------|
| DB-01 | `database/index.js:41-44` | **Migrations never run for workers/tests.** `buildServer()` calls `getDatabase()` without `migrationsDir` — tables may not exist. |
| DB-02 | `migrations/pg/` | **7 migrations (049-055) missing from PostgreSQL.** Production PG is silently diverged from SQLite dev schema. |
| DB-03 | `database/postgres.js:287-306` | **Non-atomic migration runner.** `db.exec(sql)` + separate `INSERT INTO schema_migrations` — partial failures leave broken state. |
| DB-04 | `server.js:2370-2378` | **Race condition in `incrementTrackVersion`.** Non-atomic read-after-write; concurrent callers can get same version number. |
| DB-05 | `server.js:1178-1202` | **TOCTOU in `consumeRateLimit`.** Read-then-write race allows concurrent requests to bypass rate limits. |
| DB-06 | `server.js:1494-1496` | **Gift wallet balance revert can go negative.** No floor check on `balance - amount` in idempotency reversal path. |

### Services & Providers

| ID | Location | Description |
|----|----------|-------------|
| SVC-01 | `poem-generator.js:263` | **Prompt injection via user `message` field.** `sanitizeForPrompt()` exists but is never called. User input flows directly into LLM prompt. |
| SVC-02 | `llm-provider.js:185` | **Gemini API key in URL query parameter.** Logged by all proxies, CDNs, and error tracking systems. Should use `x-goog-api-key` header. |
