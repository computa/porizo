# Music Video Feature — Saved Plan (2026-04-26)

**Status:** PAUSED after research + spike. Decision made to use sunoapi.org's
templated video endpoint. Implementation not started.

---

## TL;DR

Add a "Share as Video" feature to NowPlaying that converts a Porizo song into
a vertical 9:16 MP4 with synced lyrics + AI cover art + Porizo branding. Uses
sunoapi.org's `/api/v1/mp4/generate` endpoint. Estimated cost: ~$0.05/render.
Estimated build: ~3 days for production-ready, ~1.5 days for internal demo.

---

## Verified facts (do not re-research)

- Porizo **already uses sunoapi.org directly** for music gen (`src/providers/suno.js`, 749 lines, default per migration 046). No music-gen migration needed.
- `jobs.external_task_id` already persists the sunoapi.org `taskId`.
- `audioId` (the per-track-variant ID, also required by `/mp4/generate`) is **NOT explicitly persisted**. Available paths:
  - Capture from `sunoData[0].id` at music-gen time → forward-only
  - Extract from existing `sourceAudioUrl` pattern `cdn1.suno.ai/<UUID>.mp3` → covers historical tracks but fragile
- Output spec (verified by spike): vertical 412x732 (9:16), h264 30fps, AAC 48kHz, 326 kbps, ~5 MB for a 2-min song
- The endpoint returns a tempfile URL (`tempfile.aiquickdraw.com/r/...mp4`) that almost certainly expires — must download to S3 immediately
- The visual output: warm gold gradient + animated vinyl with AI-generated cover art + karaoke-style synced lyrics + "Generated by porizo.co" footer
- Sample artifact: `out/suno-video-spike/porizo-suno-e7a6b9f3d91c93282a5bb2e610520e7f.mp4` (still on disk)

## Rejected alternatives (do not revisit without new info)

- **Build with Remotion** — would take 2-4 weeks to match the synced-lyrics + AI-cover + animated-vinyl output. Too expensive vs $0.05/render.
- **Photo + audio APIs** (Hedra, HeyGen Avatar IV, Kling LipSync, Sync.so, D-ID) — researched, costs $0.84-$5.90 per render, user did not like options. Memory file `project_heygen_ad_blocked.md` notes existing HeyGen `/v2/avatars` issue.
- **Suno official API / Replicate Suno video** — no video endpoint available.
- **Suno consumer "Scenes" / "Hooks"** — iOS-app-only, no API.

## Implementation plan (when resumed)

### P1 — Internal demo (~1.5 days)

- [ ] **Migration**: add `video_url`, `video_status`, `video_task_id`, `external_audio_id` to `track_versions`
- [ ] **Music-gen capture**: persist `sunoData[0].id` as `external_audio_id` in `src/providers/suno.js` extraction path
- [ ] **Service**: new `src/services/share-video.js` with `generateShareVideo(trackVersionId)` → POST `/api/v1/mp4/generate`
- [ ] **Polling worker**: new queue `q.video.api` polling `/api/v1/mp4/record-info` every 5s (use existing polling pattern from `src/utils/polling.js`)
- [ ] **API routes**: `POST /tracks/:id/video` (trigger), `GET /tracks/:id/video` (status)
- [ ] **iOS button**: add "Share as Video" to `NowPlayingView.swift` and `TrackPlayerFullView.swift` (use existing `onShare` pattern)
- [ ] **Loading state**: show progress spinner / "Generating your video..." (60-90s typical)
- [ ] **Stop here**: feature flag default-off, internal team only

### P2 — User-facing (~1 day)

- [ ] **S3 persistence**: download tempfile URL → upload to `tracks/{user_id}/{track_id}/v{n}/share-video.mp4`. Handle expired URLs gracefully (the temp URL likely expires in 24-72h).
- [ ] **Cost gate**: rate-limit (e.g. 3 videos/day per user), entitlement check
- [ ] **Error code**: `R210_VIDEO_GENERATION_FAILED` + UI surface
- [ ] **Re-roll handling**: each `track_version` gets its own video (per-version isolation)
- [ ] **Old track handling**: only show "Generate Video" button when `external_task_id` AND `external_audio_id` exist; show "not available" otherwise

### P3 — Polish (~0.5 day)

- [ ] **AVPlayer preview** in iOS before share — let user see before sending
- [ ] **Telemetry**: log generation count, success rate, share rate, share-target distribution
- [ ] **Caching**: re-shares of same video are instant (read S3 URL from track_versions)
- [ ] **Webhook fallback** (optional): if polling proves wasteful, add a webhook endpoint at `POST /webhooks/suno-video` and use real callback URL instead of `httpbin.org/post`
